# Modelo de Gobernanza - Proyecto Grimoire 🏛️

Este documento describe cómo se toman las decisiones en el Proyecto Grimoire y quiénes son los responsables de su dirección.

## Modelo: Benevolent Dictator for Life (BDFL) / Dictador Benevolente

Dado que este proyecto ha nacido en una hackathon liderada por **Grimoire Project** (Grimoire Project), el modelo de gobernanza actual es de liderazgo directo.

### Roles

#### 👑 Mantenedor Líder (Lead Maintainer)
- **Responsable:** Grimoire Project
- **Poderes:** Tiene la última palabra en todas las decisiones del proyecto, incluyendo cambios de arquitectura, fusión de Pull Requests (PRs) y dirección estratégica.
- **Responsabilidad:** Garantizar la integridad del proyecto y el cumplimiento del Código de Conducta.

#### 🧙‍♂️ Colaboradores Core (Core Contributors)
- Miembros del equipo de la hackathon con permisos de escritura en el repositorio.
- Pueden revisar código y proponer cambios, pero las decisiones críticas deben ser consultadas con el Mantenedor Líder.

---

## Proceso de Toma de Decisiones

1.  **Propuestas:** Cualquier colaborador puede proponer una mejora mediante un Issue o un borrador de Pull Request (Draft PR).
2.  **Discusión:** Los cambios se debaten de forma abierta en los comentarios de GitHub o en el canal de comunicación del equipo.
3.  **Aprobación:** Solo el Mantenedor Líder (o delegados explícitos) puede aprobar la fusión de código en la rama `main`.

---

## Cómo Convertirse en Colaborador Core

Durante la hackathon, la selección es inmediata según la formación del equipo. Para contribuyentes externos post-hackathon:
- Deben haber demostrado compromiso mediante al menos 3 Pull Requests significativos.
- Deben seguir consistentemente la guía de contribución y el código de conducta.
- La invitación será extendida formalmente por el Mantenedor Líder.

---

## Resolución de Conflictos

En caso de desacuerdo técnico o personal entre colaboradores:
1. Se buscará el consenso mediante el debate técnico.
2. Si no hay acuerdo, el Mantenedor Líder tomará una decisión vinculante para mantener el progreso del proyecto.

---
*Este modelo de gobernanza es ligero para permitir el desarrollo rápido durante eventos de innovación, pero escala hacia un modelo más participativo a medida que la comunidad crece.*
